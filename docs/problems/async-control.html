Control Async Workflows

Use @jdsalasc/solvejs-async to add retries, promise timeouts, and concurrency limits for production jobs.

Cookbook Navigation

Recipe

import { retry, timeout, pMap } from "@jdsalasc/solvejs-async";

const payload = await retry(
  () => timeout(fetch("https://api.example.com/items").then((r) => r.json()), 3000),
  { retries: 2, delayMs: 200, backoffFactor: 2 }
);

const ids = await pMap(payload.items, async (item) => item.id, { concurrency: 4 });

Why this works: you get predictable retry/timeout behavior and controlled parallelism without external runtime dependencies.
