Transform Arrays and Run Safe Math

Use @jdsalasc/solvejs-list and @jdsalasc/solvejs-numbers for robust collection pipelines and business calculations.

Recipe

import { uniqueBy, groupBy, sortBy } from "@jdsalasc/solvejs-list";
import { toNumber, safeDivide, percentChange } from "@jdsalasc/solvejs-numbers";

const rows = [{ id: "a", team: "x", value: "120" }, { id: "a", team: "x", value: "120" }, { id: "b", team: "y", value: "80" }];
const uniqueRows = uniqueBy(rows, (r) => r.id);
const grouped = groupBy(uniqueRows, (r) => r.team);
const sorted = sortBy(uniqueRows, (r) => Number(r.value), "desc");

const current = toNumber(sorted[0].value);
const previous = toNumber(sorted[1].value);
if (current !== null && previous !== null) {
  percentChange(current, previous);
  safeDivide(current, 0, 0);
}

Why this works: utility composition keeps transforms explicit and reduces hidden edge cases.

Production Notes

  • For financial values, apply explicit rounding boundaries (for example with roundTo) to control floating-point drift.
  • For large list workloads (10k/100k), run npm run benchmark and validate uniqueBy, groupBy, and sortBy against representative data shapes.
