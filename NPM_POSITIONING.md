# SolveJS npm Positioning Guide

Use this file to keep package messaging and SEO terms consistent.

## Brand Promise

`SolveJS: zero-dependency utilities that solve real JavaScript and TypeScript production problems.`

## Package Positioning

- `@jdsalasc/solvejs`
  - Position: all-in-one utility suite.
  - Search intents: `javascript utility library`, `typescript utility library`.
- `@jdsalasc/solvejs-date`
  - Position: strict parsing + UTC-safe date helpers.
  - Search intents: `javascript date format`, `typescript date parse`.
- `@jdsalasc/solvejs-string`
  - Position: practical string normalization and formatting.
  - Search intents: `slugify javascript`, `title case typescript`, `remove html tags js`.
- `@jdsalasc/solvejs-list`
  - Position: fast, predictable list transforms.
  - Search intents: `group array by key javascript`, `partition array typescript`.
- `@jdsalasc/solvejs-regex`
  - Position: battle-tested regex patterns.
  - Search intents: `email regex javascript`, `uuid regex typescript`, `ipv4 regex js`.
- `@jdsalasc/solvejs-constants`
  - Position: reusable constants for app logic.
  - Search intents: `http methods constants`, `file size constants`.
- `@jdsalasc/solvejs-numbers`
  - Position: safe math for product/business code.
  - Search intents: `percent change javascript`, `safe divide typescript`.
- `@jdsalasc/solvejs-validators`
  - Position: structured form validation with result codes.
  - Search intents: `validate phone number javascript`, `name validator typescript`.

## Release Copy Template

`SolveJS vX.Y.Z adds [feature list] to solve [specific pain points]. All packages stay zero-dependency and ship ESM, CJS, and TypeScript types.`

## Keyword Policy

- Keep 8 to 14 high-intent keywords per package.
- Always include both `javascript` and `typescript`.
- Include at least 3 problem-specific terms per package.
